# AGENTS.md

この指示書は頼れるCodexへのおねがいです！いっぱい守ってね！

## このアプリの仕様と目的

specification.md を読んでね！

## セッション開始時に毎回やること

- まずはspecification.md を読んでね！
- `issues/` ディレクトリの最新の日付のものを読んでね。
- `task_report/` に、これまでの作業ログがあるから、特に最新分を読んでね！  
  - 最新分だけで理解しきれないことがあったら、それ以前のものも読んでね！

## 作業中のルール

- 基本的には specification.md にまとめてあるけど、わからないことはまずユーザーに確認してね！
- commitは「なんかしたほうがいいな」っていう時はこまめにしていいよ。  
  ただし日本語コメントで、何をしたかと、Claudeの作業だよっていうのはわかるようにしてね。

## 作業後のルール

その日の最後は `task_report/` に、yymmdd.md でファイルをつくって、その日の進捗や翌日以降に残ったタスクを書いてね。  
もし今日のファイルがすでにあった場合は、そのファイルに追記でOKだよ。  
ちなみに `issues/` と `task_report/` は、外に見せるのがはずかしいのと、アプリとしてのファイルじゃないから .gitignore してるよ。

## コードレビューをお願いしたときのルール

コードレビューへのチェック内容は、`task_report/` 直下に、review_yymmdd.md を作って、
気になったことリストを重要箇所の順番に書き出してね。  
Codexに直してもらうか、Claudeちゃんに手戻りおねがいするかはユーザーが決めるよ。
